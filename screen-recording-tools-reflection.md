🎥 Learn about screen recording tools
🎯 Goal
Understand how to capture high-quality screen recordings and screenshots for bug reporting.

❓ Why is this important?
Clear screen recordings and screenshots make it easier for developers to understand and reproduce issues, leading to faster bug fixes.

🔍 Research & Learn

**What are the best free and paid screen recording tools for Windows/macOS?**
Some of the best screen recording tools include both free and paid options. On Windows, common tools are OBS Studio, ShareX, Xbox Game Bar, and Loom. On macOS, tools like QuickTime Player, Loom, and OBS Studio are widely used. OBS Studio is powerful and free, while Loom is popular for quick recordings and easy sharing. ShareX is also useful because it supports both screen recording and screenshots.

**How do you record a specific part of the screen instead of the entire display?**
Most screen recording tools allow you to capture only a specific part of the screen instead of the entire display. When starting a recording, you can usually choose between full screen, a specific window, or a custom region. Selecting a smaller area helps focus on the bug and reduces file size.

**What are best practices for including screen recordings in bug reports?**
Best practices for including screen recordings in bug reports include clearly showing the steps that lead to the issue. The recording should start slightly before the action that triggers the bug and end after the problem appears. It is helpful to move the cursor slowly so developers can see each step clearly. If possible, include narration or text in the issue description explaining what is happening in the video.

**How can you reduce video file sizes while maintaining good quality?**
To reduce video file size while keeping good quality, you can record only the necessary screen area, keep the recording short, lower the frame rate slightly, or compress the video after recording. Some tools automatically compress videos when uploading or exporting them.

📝 Reflection

**What key information should you include when submitting a screen recording of a bug?**
When submitting a screen recording of a bug, it is important to include clear steps that lead to the issue, the moment the bug occurs, and any visible error messages. The recording should also match the written description in the bug report so developers can easily follow what happened.

**How can you ensure sensitive information is not accidentally shared in a recording?**
To ensure sensitive information is not accidentally shared, testers should close unrelated tabs or applications before recording and avoid showing personal data, passwords, or confidential information. If needed, sensitive areas can be blurred or cropped before sharing the video.

**If a bug only happens during fast interactions, how would you record and describe it effectively?**
If a bug happens only during very fast interactions, I would record the screen while performing the actions several times until the issue appears. I would also describe the actions clearly in the bug report, such as “rapidly clicking the button multiple times” or “switching between screens quickly.” If necessary, the video can be replayed in slow motion to make the problem easier to observe.
