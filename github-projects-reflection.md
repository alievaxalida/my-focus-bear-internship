📝 Get familiar with GitHub Projects
🎯 Goal
Understand how GitHub Projects is used to track issues and manage QA tasks.

❓ Why is this important?
GitHub Projects helps QA interns organise and prioritise tasks, track progress, and collaborate effectively with the development team.

🔍 Research & Learn

**What are GitHub Projects, and how do they work?**
GitHub Projects is a project management tool inside GitHub that helps teams organize and track work such as bugs, tasks, and feature requests. It usually works like a board with columns where issues move through different stages of work. Teams create issues to describe tasks or problems, and these issues are added to the project board so everyone can see progress and responsibilities.

**How are issues created, assigned, and moved through different project columns (To Do, In Progress, Done)?**
In GitHub Projects, issues are created when someone reports a bug, requests a feature, or adds a task. After creating an issue, it can be assigned to a team member who is responsible for working on it. The issue is then placed in a project column such as “To Do,” “In Progress,” or “Done.” As work progresses, the issue is moved between columns to reflect its current status. This helps the whole team understand what is planned, what is currently being worked on, and what has been completed.

**What are labels, milestones, and assignees in GitHub issues?**
Labels, milestones, and assignees help organize issues more clearly. Labels categorize issues, for example bug, enhancement, UI, or high priority. Milestones group issues that belong to the same release or development goal. Assignees show which person is responsible for resolving the issue. These features make it easier to manage large numbers of tasks and track progress.

**How can you use GitHub Projects to keep track of testing progress?**
QA testers can use GitHub Projects to track testing progress by creating issues for discovered bugs, labeling them correctly, and linking them to the appropriate milestone or release. Testers can also update issue status as testing progresses and verify when a bug is fixed. This helps maintain visibility and coordination with the development team.

📝 Reflection

**How would you create a new issue for a bug found during testing?**
If I found a bug during testing, I would create a new issue with a clear title describing the problem. In the description I would include steps to reproduce the bug, the expected result, the actual result, and environment details such as device, OS, or app version. I would also attach screenshots, logs, or screen recordings if available.

**How can you prioritise issues in a GitHub Project?**
To prioritize issues in a GitHub Project, I would use labels such as high, medium, or low priority, and focus first on issues that block core functionality or affect many users. Critical bugs should be addressed before minor cosmetic issues.

**What are some advantages of using GitHub Projects for tracking QA tasks?**
Some advantages of using GitHub Projects for QA tasks include clear visibility of work, better organization of bugs and tasks, easier collaboration between QA and developers, and the ability to track progress over time. It helps ensure that issues are not forgotten and that the team can manage testing and development more efficiently.
