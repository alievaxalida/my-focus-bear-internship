📝 Reflection

**How would you report a UI glitch vs. a critical login failure differently?**
For a UI glitch (like a misaligned button), I would focus more on screenshots and visual details. The priority would be lower. For a critical login failure, I would prioritize it as "Critical" and provide clear, step-by-step instructions. I would also include error logs or API responses because this bug stops users from using the app entirely.

**If a developer dismisses a bug as "not important," how would you advocate for fixing it?**
I would explain the impact on the user. I would describe a real-life scenario: "If a user cannot see this button, they might get frustrated and delete the app." Using facts and user perspective helps convince the developer that the bug is worth fixing.

**How can screenshots, logs, and screen recordings improve your bug reports?**
They save time and reduce confusion. Screenshots show the visual problem, screen recordings show the exact steps to reproduce it, and logs show what is happening behind the scenes (the code level). This makes it much easier for the developer to find the cause without asking me too many questions.
