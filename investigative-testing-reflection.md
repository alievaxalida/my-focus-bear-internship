📝 Reflection

**What questions would you ask when testing a new Focus Bear feature for the first time?**
I would ask: "Who is this feature for?", "What problem is it trying to solve?", and "What is the worst thing that could happen if this feature fails?" I would also ask "What if...?" questions, like "What if the user has no internet while using this?"

**How can uncertainty and curiosity make you a better tester?**
Uncertainty makes me double-check things instead of assuming they work. Curiosity drives me to try different paths that a normal user might take. A curious tester finds the bugs that are hidden deep in the software's logic.

**What techniques could you use to dig deeper when a bug is difficult to reproduce?**
I would use Variable Change (changing one thing at a time, like device orientation or network speed). I would also check the Logs and Console to see what's happening in the background. Another technique is to look for patterns—does it only happen on a specific screen or after a specific action?
